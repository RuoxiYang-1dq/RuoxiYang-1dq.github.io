## Python实战：完整接入示例

### 代码框架与关键参数

下面是一份可以直接复制运行的完整 Python 代码。关键改动只有两行：`base_url` 和 `api_key`。

python
import openai

# 1. 关键配置
base_url = "https://www.qianjuai.com/v1"
api_key = "sk-你的密钥"  # 替换成你从千聚控制台复制的密钥

# 2. 初始化客户端
client = openai.OpenAI(
    base_url=base_url,
    api_key=api_key
)

# 3. 构建请求
model = "claude-sonnet-4-5"  # 千聚上的模型名，确认无误
messages = [
    {
        "role": "user",
        "content": "请用Python写一个快速排序算法，并附带详细注释。"
    }
]

try:
    # 4. 发送请求（非流式）
    response = client.chat.completions.create(
        model=model,
        messages=messages,
        max_tokens=4096,
        temperature=0.7
    )
    
    # 5. 获取并输出回复
    reply = response.choices[0].message.content
    print(reply)

except Exception as e:
    print(f"请求出错：{e}")


**代码解析：**

- `base_url = "https://www.qianjuai.com/v1"` 是把所有请求定向到千聚的网关，由它替你去连接 Anthropic 的海外接口。
- `model` 参数的值 `"claude-sonnet-4-5"` 是千聚API中转站为 Claude Sonnet 4.5 分配的模型标识。如果不确定模型名，可以在官方文档或控制台的“模型列表”里查询。
- `max_tokens` 控制最大输出长度，Claude Sonnet 4.5 支持最高 8192 个输出 token，根据你需要设置即可。

### 启用流式输出（推荐）

如果你想让 AI 的回答像 ChatGPT 一样逐字出现，而不是等全部生成完才一次性显示，可以启用流式模式。修改代码中的调用部分：

python
response_stream = client.chat.completions.create(
    model=model,
    messages=messages,
    max_tokens=4096,
    temperature=0.7,
    stream=True  # 启用流式
)

# 逐块输出
for chunk in response_stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="", flush=True)

## 常见问题与调试建议

### Q1: 报错 `401 Authentication failed`

答：检查你的 API Key 是否已过期，或者是否粘贴时漏了字符。可以去控制台重新生成一个试试。

### Q2: 报错 `404 model not found`

答：模型名不匹配。登录控制台查看“模型列表”，确认 Claude Sonnet 4.5 的准确名称。有时候模型命名可能包含版本号（如 `claude-sonnet-4-5-20250401`），以列表显示为准。

### Q3: 响应太慢或超时

答：确认本机网络环境是否稳定。如果使用的是企业内网或校园网，可以尝试切换网络环境。另外，在 `client.chat.completions.create` 的参数里可以增加 `timeout=60` 参数延长等待时间。

### Q4: 如何切换不同模型版本？

答：很简单。同一套代码，只需要改 `model` 参数里的字符串。把 `claude-sonnet-4-5` 换成 `gpt-4o`、`gemini-2.5-pro`、`deepseek-r1`，都能跑——千聚API中转站一个接口兼容所有模型。

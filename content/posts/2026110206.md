## 部署有多简单——真的是一行命令

这才是这个方案最迷人的地方。部署的核心就是Docker，你只要在你的服务器上执行下面这一行命令：

bash
docker run --name one-api -d --restart always -p 3000:3000 -e TZ=Asia/Shanghai -v /home/ubuntu/data/one-api:/data justsong/one-api


**这行命令干了什么？**

1.  **拉取镜像**：自动下载官方的 `one-api` Docker镜像。
2.  **启动容器**：在后台运行一个名为 `one-api` 的容器。
3.  **端口映射**：把你服务器的 `3000` 端口映射到容器内的 `3000` 端口。之后你就可以通过 `http://你的服务器IP:3000` 访问后台管理界面了。
4.  **数据持久化**：把容器内的数据文件夹 `/data` 映射到你本机的 `/home/ubuntu/data/one-api` 目录。这样即使你删除了容器，你的配置和Key数据也不会丢。
5.  **时区设置**：确保日志时间正确。

**部署完成之后，你只需要做三步：**

1.  **登录后台**：在浏览器打开 `http://你的服务器IP:3000`。默认账号是 `root`，密码是 `123456`（请第一时间修改）。
2.  **添加模型**：在后台的“渠道”管理页面，根据你的实际情况添加模型。比如你有一个DeepSeek的API Key，就添加一个“DeepSeek”的渠道，并填上你获取的API Key和Base URL。
3.  **创建令牌**：在“令牌”页面创建一个新的访问令牌（Token）。这个Token就是你之后所有客户端要用的Key。

**这样，一个属于你自己的、支持500+模型的OpenAI兼容接口就搭建完成了。**

## 为什么用Docker？它有什么优势

Docker是这个方案里最核心的“一键”按钮。如果你以前没接触过，可以把Docker想象成一个轻量化的虚拟机，它能保证“你的环境”和“别人的环境”一模一样，从而避免了各种由于系统差异导致的“在我电脑上能跑”的问题。

具体到我们这个场景：

1.  **一键化**：你不需要先装个Python3.10，再装个Node.js，再装个MySQL，再改配置文件。一条Docker命令干完了所有事情。
2.  **高可用性**：Docker提供了`--restart always`参数，意味着即使你的服务器意外重启了，你的API接口也会自动启动，不需要你手动登录去点“运行”。
3.  **隔离性好**：你的API服务被封装在一个独立容器里，不会影响你服务器上其他正在运行的服务（比如博客、数据库等）。
4.  **易于迁移**：如果以后你换服务器了，只需要把整个 `/home/ubuntu/data/one-api` 文件夹和Docker命令一拷贝，在新机器上运行一遍，数据完全不变。

所以，Docker在这里不是锦上添花，它就是这个方案“一键部署”的保证。

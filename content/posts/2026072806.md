### 示例一：最简洁的Grok3对话调用

我推荐你使用Java 11及以上版本内置的 `java.net.http.HttpClient`，这样不需要引入第三方依赖，代码简洁干净。下面是一个完整的可运行示例：

java
import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;
import java.net.http.HttpRequest.BodyPublishers;
import java.net.http.HttpResponse.BodyHandlers;

public class Grok3ChatExample {
    public static void main(String[] args) throws Exception {
        // 1. 创建HttpClient客户端
        HttpClient client = HttpClient.newHttpClient();

        // 2. 构建请求体（JSON格式）
        String jsonBody = """
                {
                    "model": "grok-3",
                    "messages": [
                        {
                            "role": "user",
                            "content": "你好，请用中文简短地介绍一下你自己。"
                        }
                    ],
                    "max_tokens": 200,
                    "temperature": 0.7
                }
                """;

        // 3. 构建HTTP POST请求
        HttpRequest request = HttpRequest.newBuilder()
                .uri(URI.create("https://www.qianjuai.com/v1/chat/completions"))
                .header("Authorization", "Bearer 你的API Key在这里") // 重要：请替换为你的真实API Key
                .header("Content-Type", "application/json")
                .POST(BodyPublishers.ofString(jsonBody))
                .build();

        // 4. 发送请求并获取响应
        HttpResponse<String> response = client.send(request, BodyHandlers.ofString());

        // 5. 打印响应结果
        System.out.println("响应状态码: " + response.statusCode());
        System.out.println("响应内容: " + response.body());
    }
}


**核心要点讲解：**
- **base_url（基础地址）**：必须使用 `https://www.qianjuai.com/v1`，这是[千聚api聚合平台](https://www.qianjuai.com/)的标准端点。
- **Authorization（鉴权头）**：格式为 `Bearer 你的API Key`，请务必去掉引号，直接拼接你的Key。
- **model（模型名称）**：使用 `grok-3` 作为模型名称。
- **messages（消息列表）**：这是OpenAI标准格式，`role` 为 `user` 表示用户输入，`content` 是具体的问题内容。

- 使用try-catch块捕获所有可能的异常，保证程序稳定性。

### 示例三：流式输出（Stream）实现实时对话

如果你想让Grok3像ChatGPT一样一个字一个字地输出，就需要使用流式（Stream）模式。[千聚api聚合平台](https://www.qianjuai.com/)完美支持Server-Sent Events（SSE）。下面是一个简单的实现：

java
import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;
import java.net.http.HttpRequest.BodyPublishers;

public class Grok3StreamExample {
    public static void main(String[] args) throws Exception {
        String apiKey = "你的API Key在这里"; // 请替换

        String jsonBody = """
                {
                    "model": "grok-3",
                    "messages": [
                        {"role": "user", "content": "请用中文写一首关于夏天的短诗。"}
                    ],
                    "stream": true
                }
                """;

        HttpClient client = HttpClient.newHttpClient();
        HttpRequest request = HttpRequest.newBuilder()
                .uri(URI.create("https://www.qianjuai.com/v1/chat/completions"))
                .header("Authorization", "Bearer " + apiKey)
                .header("Content-Type", "application/json")
                .POST(BodyPublishers.ofString(jsonBody))
                .build();

        // 流式响应处理
        client.send(request, HttpResponse.BodyHandlers.ofLines())
                .lines()
                .filter(line -> line.startsWith("data: "))
                .map(line -> line.substring(6).trim())
                .filter(data -> !data.equals("[DONE]"))
                .forEach(data -> {
                    // 解析每个data块，提取content
                    int start = data.indexOf("\"content\":\"") + 11;
                    if (start > 10) {
                        int end = data.indexOf("\"", start);
                        String content = data.substring(start, end)
                                .replace("\\n", "\n")
                                .replace("\\t", "\t");
                        System.out.print(content);
                    }
                });
    }
}


这个流式示例代码会逐块打印Grok3生成的内容，用户体验更好。你可以在循环中添加一个小延迟，或者将内容实时显示在GUI界面上。

## 进阶技巧与常见问题

### 如何调整Grok3的回答风格？

通过修改请求参数可以控制回答风格：
- **temperature**：控制创造性的参数，值范围0-2。0.2以下回答更确定、严谨；0.8以上更有创意、更随机。简单回答建议用0.3，创意写作建议用0.8。
- **max_tokens**：控制回答的最大长度，单位是token。一般对话设为500-1000即可。
- **top_p**：另一种控制多样性的参数，通常与temperature二选一使用。

### 常见报错及解决方案

**错误1：401 Unauthorized**
原因：API Key错误或未正确设置。
解决：检查你的Headers中 `Authorization` 字段，确保格式为 `Bearer sk-xxxxx`，且没有多余空格。

**错误2：404 Not Found**
原因：请求的URL地址错误。
解决：确认你使用的是 `https://www.qianjuai.com/v1/chat/completions`，不要遗漏 `/v1` 路径前缀。

**错误3：400 Bad Request**
原因：请求体格式错误。
解决：检查JSON格式是否正确，特别是花括号、引号、逗号的配对。

**错误4：502 Bad Gateway**
原因：网络波动或千聚服务器临时问题。
解决：稍等几秒重试，或者检查你的网络环境是否稳定。

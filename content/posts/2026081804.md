### 2. 基础调用：一句话生成内容

看下面的代码，这几乎是你能写出的最简调用方式，完美诠释了“低代码接入”：

javascript
const OpenAI = require('openai');

const client = new OpenAI({
  apiKey: '你的API Key',
  baseURL: 'https://www.qianjuai.com/v1', // 关键：换成千聚的地址

async function main() {
  const chatCompletion = await client.chat.completions.create({
    model: 'gpt-4o', // 你想用的模型

  });
  console.log(chatCompletion.choices[0]?.message?.content);
}

main();


你看，只需要声明一个OpenAI客户端，指定模型和消息，就能得到回复。没有额外的认证逻辑，没有参数拼接，就是纯粹的“声明式”编程。

实际应用中，我们往往需要支持多轮对话。在低代码模式下，你只需要维护一个`messages`数组，把历史对话传进去就行：

整个调用逻辑非常清晰。你只需要专注于“消息”本身，而不需要关心底层的HTTP请求如何构建，这是“低代码”最爽的地方。

### 4. 流式输出（SSE）示例

如果你的应用需要打字机效果，比如AI逐字输出回答，同样只需要加一句配置：

javascript
async function streamChat() {
  const client = new OpenAI({
    apiKey: '你的API Key',
    baseURL: 'https://www.qianjuai.com/v1',
  });

  const stream = await client.chat.completions.create({
    model: 'gpt-4o',
    messages: [{ role: 'user', content: '写一首关于Node.js的诗' }],
    stream: true, // 开启流式输出
  });

  for await (const chunk of stream) {
    process.stdout.write(chunk.choices[0]?.delta?.content || '');
  }
}

streamChat();


看，只需要增加一个`stream: true`参数，你就获得了实时响应。底层的事件流解析、数据缓冲，全部由库和[千聚ai聚合平台](https://www.qianjuai.com/)处理好了。你只管拿数据。

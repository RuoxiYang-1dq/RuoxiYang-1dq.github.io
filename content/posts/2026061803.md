## 常见报错处理：我已帮你踩过坑

即使步骤如此简单，但新手在接入时依然可能遇到几个常见报错。我把它们整理出来，你遇到任何一个，直接按下面的方案排查即可，不用再花时间百度或问客服。

**1. 报错 `404 Not Found` 或 `Invalid Model`**
- **原因**： `model` 字段填写的模型名称不对。千聚平台上的 Grok4 模型ID不是 `grok-4` 也不是 `grok-4-xx`，而是 **`grok-4-0824`**。
- **解决方法**： 在代码中将 `model` 参数的值严格改为 `grok-4-0824`。你可以在千聚的控制台页面查看所有支持模型的准确ID列表。

**2. 报错 `401 Authentication Error`**
- **原因**： API Key 填写错误、Key 已过期、或者你在代码中不小心将 `api_key` 写成了别的变量名。
- **解决方法**：
  - 重新检查你的 API Key 是否复制正确，注意不要复制到前面的空格。
  - 确认你的 Key 在千聚控制台里是「启用」状态，且余额不为零。
  - 确保代码中的参数名字是 `api_key` 或者 `openai.api_key`。

**3. 报错 `RateLimitError` 或 `429 Too Many Requests`**
- **原因**： 你账户的并发请求达到了千聚平台设定的限制（通常免费额度或者新账户有限制，但标准账户一般无并发限制）。
- **解决方法**：
  - 检查你的计划，如果是免费用户，可尝试降低并发请求数或稍后再试。
  - 如果确认账户是标准账户，且已经充值，请在千聚控制台查看是否有「并发限制」设置，或者联系客服提高限制。

**4. 报错 `Connection Error` 或 `Timeout`**
- **原因**： 这是最常见的「卡脖子」问题。国内网络环境不稳定，无法直接解析或连接到 xAI 的海外服务器。你虽然用了千聚，但如果你的代码里没有正确使用千聚的 `base_url`，或者你本地网络环境有代理冲突，依然可能触发此错误。
- **解决方法**：
  - 再次确认你的 `base_url` 设置为 `https://www.qianjuai.com/v1`。
  - 如果你本地开启了梯子，请暂时关闭它，因为千聚已经是国内直连，不需要代理。开代理反而可能走错路。
  - 如果你是使用 Cursor、LobeChat、Cherry Studio 等第三方客户端，请确认你配置的 API 地址也正确填写为了千聚的地址。

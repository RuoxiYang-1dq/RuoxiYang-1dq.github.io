## 踩坑最多的地方：避坑指南

我在测试过程中踩过几个坑，列举出来帮你跳过。

**坑1：API Key写错了或者被盗用了**

很多新手把Key直接写在代码里，然后不小心上传到GitHub公开仓库。《黑客》扫到Key后能直接刷光你的额度。

**避坑方案：使用环境变量**

写代码时不要硬编码Key，而是从环境变量读取：

python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.getenv("QIANJU_API_KEY"),  # 在系统环境变量里设置
    base_url="https://www.qianjuai.com/v1"
)


设置环境变量的方式（Linux/Mac）：

bash
export QIANJU_API_KEY="sk-你的Key"


Windows用户可以用`set`命令，或者在代码里用python-dotenv库从`.env`文件读取。

**坑2：base_url写错了或用了旧版地址**

有人把base_url写成了`https://api.openai.com/v1`，自然调不通，因为DeepSeek-R1不在OpenAI服务器上。

**正确写法：**


https://www.qianjuai.com/v1


注意最后面的`/v1`不能少，少了会返回404错误。

**坑3：模型名字写错了，导致调用失败**

在千聚平台里，DeepSeek-R1对应的模型ID是`"deepseek-r1"`，不要写成`"DeepSeek-R1"`或`"deepseek-r1-0310"`。大小写和连字符必须完全一致。

如果还是不确定，可以登录千聚后台查看“模型列表”，它会列出当前所有可用的模型名称。

**坑4：超时问题，尤其是长文本生成时**

DeepSeek-R1在生成复杂推理时可能耗时很长，如果客户端没有设置超时时间，连接会被中断。

**解决方案：给客户端设置超时**

python
client = OpenAI(
    api_key=os.getenv("QIANJU_API_KEY"),
    base_url="https://www.qianjuai.com/v1",

## 进阶：流式输出实现打字机效果

普通调用是等全部结果生成完成后一起返回，流式输出则是边生成边输出，用户体验更好。

python
from openai import OpenAI

client = OpenAI(
    api_key="sk-你的Key",
    base_url="https://www.qianjuai.com/v1"
)

stream = client.chat.completions.create(
    model="deepseek-r1",
    messages=[{"role": "user", "content": "给我讲一个关于程序员和机器学习的故事"}],
    stream=True,  # 关键参数
)

for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="", flush=True)


运行这个脚本，你会看到文字一个字一个字地打印出来，就像ChatGPT官网的效果。流式输出对实时交互场景非常有用。

## 完整项目实践：用DeepSeek-R1搭建一个智能助手

给你一个更完整的示例：一个简单的命令行问答助手，支持多轮对话和流式输出。

python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.getenv("QIANJU_API_KEY"),
    base_url="https://www.qianjuai.com/v1"
)

def chat_with_deepseek():
    print("DeepSeek-R1 问答助手已启动（输入exit退出）")
    messages = []  # 保存对话历史

    while True:
        user_input = input("\n你的问题：")
        if user_input.lower() == "exit":
            break

        messages.append({"role": "user", "content": user_input})

        # 流式输出
        stream = client.chat.completions.create(
            model="deepseek-r1",
            messages=messages,
            stream=True
        )

        print("R1回复：", end="")
        reply = ""
        for chunk in stream:
            if chunk.choices[0].delta.content:
                content = chunk.choices[0].delta.content
                print(content, end="", flush=True)
                reply += content

        messages.append({"role": "assistant", "content": reply})

if __name__ == "__main__":
    chat_with_deepseek()


这个脚本你直接复制到本地运行，就是一个可以持续对话的AI助手。所有历史消息都会参与上下文，所以它能记得你之前聊过什么。

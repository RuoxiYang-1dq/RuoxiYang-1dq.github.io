## RAG到底是什么？简单理解就够了

RAG（Retrieval-Augmented Generation，检索增强生成）说白了就是：**给大模型配一本“课外书”**。

平时你问大模型“今年的财报怎么样”，它可能只能凭训练数据回答，无法知道最新信息。RAG的做法是：先从你的私域知识库（比如企业文档、产品手册、聊天记录）里把最相关的内容搜出来，再把这些内容“塞”给大模型，让它基于这些实时信息来回答。

这套流程用代码写起来其实就三步：
1. 把文档切分成小块，做向量化存储
2. 用户提问时，从向量库里搜到最相关的几块内容
3. 把搜索结果拼到Prompt里发给大模型，拿到最终答案

import openai
from chromadb import Client
from chromadb.utils import embedding_functions

# 配置千聚AI大模型聚合站
client = openai.OpenAI(
    api_key="YOUR_QIANJU_API_KEY",  # 替换成你的Key

    "年假必须在当年内使用，不能累积到下一年。",
    "申请年假需要提前三天在OA系统提交申请。",
    "法定节假日不占用年假额度。"

embedding_fn = embedding_functions.OpenAIEmbeddingFunction(
    api_key="YOUR_QIANJU_API_KEY",
    model_name="text-embedding-ada-002",
    api_base="https://www.qianjuai.com/v1"

    name="employee_handbook",
    embedding_function=embedding_fn
)

# 添加文档
collection.add(
    documents=documents,
    ids=["0", "1", "2", "3"]
)

# 3. 用户提问
question = "新员工入职多久可以休年假？"

# 4. 检索最相关内容
results = collection.query(
    query_texts=[question],
    n_results=2
)
relevant_docs = results["documents"][0]

# 5. 组装Prompt并调用大模型
prompt = f"""基于以下信息回答问题：
{chr(10).join(relevant_docs)}

问题：{question}
回答："""

response = client.chat.completions.create(
    model="gpt-4o-mini",  # 你可以换成其他模型，比如 qwen-plus
    messages=[
        {"role": "system", "content": "你是公司HR助手。"},
        {"role": "user", "content": prompt}
    ]
)

print(response.choices[0].message.content)


跑完这段代码，你的控制台就会打印出类似这样的答案：

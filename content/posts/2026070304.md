## 核心步骤：Java 调用 Qwen-VL 示例

我们将使用官方的 `openai-java` 客户端库，因为它天然兼容[千聚ai聚合平台](https://www.qianjuai.com/)的接口。你不需要额外学习 Qwen 独有的 SDK，代码改动量极小。

<dependency>

> 注意：这里用 `com.theokanning.openai-gpt3-java` 这个库，它支持最新的 Chat Completions 接口，非常稳定。

### 2. Java 核心调用代码

创建一个简单的 Java 类，我们一步一步写。

java
import com.theokanning.openai.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import com.theokanning.openai.completion.chat.ChatMessageRole;
import java.util.Arrays;
import java.util.List;

public class QwenVlExample {
    public static void main(String[] args) {
        // 1. 设置你的API Key和接入地址
        String apiKey = "sk-你的真实密钥";
        String baseUrl = "https://www.qianjuai.com/v1";

        // 2. 构建OpenAiService对象，传入自定义baseUrl
        OpenAiService service = new OpenAiService(baseUrl, apiKey);

        // 3. 构建多模态请求消息（支持图片）
        // 注意：Qwen-VL接受base64编码的图片字符串
        String imageBase64 = "iVBORw0KGgoAAAANSUh..."; // 你的图片base64编码
        String imageUrl = "data:image/png;base64," + imageBase64;

        // 4. 构造消息体（系统提示 + 用户消息）
        ChatMessage systemMessage = new ChatMessage(ChatMessageRole.SYSTEM.value(), "你是一个图像分析专家");
        ChatMessage userMessage = new ChatMessage(ChatMessageRole.USER.value());
        userMessage.setContent(List.of(
            Map.of("type", "image_url", "image_url", Map.of("url", imageUrl)),
            Map.of("type", "text", "text", "请描述这张图片的内容。")
        ));

        List<ChatMessage> messages = Arrays.asList(systemMessage, userMessage);

        // 5. 创建请求
        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("gpt-4o") // 这里填写你开通了哪个模型。千聚支持gpt-4o兼容Qwen等。
                .messages(messages)
                .maxTokens(500)
                .temperature(0.7)
                .build();

        // 6. 发起调用并打印结果
        service.createChatCompletion(request)
                .getChoices()
                .forEach(choice -> System.out.println("模型回答: " + choice.getMessage().getContent()));
    }
}

## 零报错核心原则：对照表

如果你在运行过程中遇到报错，下面是常见问题及对照表。大部分报错都不是代码逻辑问题，而是环境和参数不对。

| 报错现象 | 原因 | 解决方案 |
| --- | --- | --- |
| `java.net.ConnectException: Connection refused` | 网络不通，或域名解析失败。 | 检查是否在能访问国内网络的环境。用 `ping www.qianjuai.com` 测试连通性。 |
| `400 Bad Request` | 请求体格式错误（常见为图片字段拼写或结构不符）。 | 确认 `image_url` 字段名正确，且 `Map` 嵌套结构完全符合OpenAI多模态标准。 |
| `401 Unauthorized` | API Key错误或过期。 | 在[千聚ai聚合平台](https://www.qianjuai.com/)控制台重新生成Key，并检查代码中拼写。 |
| `model_not_found` 或 `404 Not Found` | 填写的模型名未开通或不支持。 | 登录[千聚ai聚合平台](https://www.qianjuai.com/)，查看“限时特价分组”中已开通的模型。 |
| `SSLHandshakeException` | Java环境SSL证书问题，或系统时间不正确。 | 更新Java证书信任库，或设置系统时间自动同步。 |
| 空指针或`NullPointerException on base64` | 图片base64字符串为空或格式错误。 | 确保图片base64编码正确，且不包含换行符。建议用在线工具验证。 |
| `java.lang.IllegalArgumentException: Unsupported content type` | 消息内容的格式不匹配（比如把文本直接当成了图片）。 | 确保 `userMessage.setContent()` 参数是 `List<Map>` 类型，而不是纯字符串。 |
| 返回内容为空或乱码 | token数量超限或编码问题。 | 调整 `maxTokens` 参数，或检查response的encoding。 |

> **一个小经验**：如果你遇到了 `400 Bad Request`，90%的可能是图片字段没写对格式。确保你的代码中，`image_url` 的 `Map` 层级是 `{"type": "image_url", "image_url": {"url": "data:image/png;base64,..."}}`。严格遵循这个结构。

## 进阶技巧：提高成功率的小修改

1. **使用重试机制**：在API调用外层加一个简单的重试循环，比如失败后等待2秒再试一次：

java
int retries = 3;
while (retries-- > 0) {
    try {
        service.createChatCompletion(request)...
        break;
    } catch (Exception e) {
        if (retries == 0) throw e;
        Thread.sleep(2000);
    }
}


2. **图片压缩**：如果图片太大，base64字符串会很长，容易超token。建议先把图片压缩到1MB以下再编码。可以用 `javax.imageio.ImageIO` 配合 `new ByteArrayOutputStream()` 实现。

## 从高额账单到每月 99 元的调优路线

光接入还不够，怎么真正把费用压下来？我从实际踩坑中总结了三步。

### 第一步：选择正确的分组

千聚平台提供了多个分组，但 GLM-4.5 最佳选择是**限时特价分组**。这个分组覆盖了 DeepSeek、Qwen、Gemini 以及 GLM 系列，费率最低只有官方的 0.6 倍。充进去 1 块钱，能当官方 1.7 块钱花。

在千聚后台创建 API Key 时，确保选择限时特价分组。这一步直接决定你的基础成本。

### 第二步：控制 Token 消耗

GLM-4.5 的上下文窗口很大（128k），但大上下文意味着高费用。实际开发中不需要每次都塞满整个历史。

我习惯的做法是：

- 设定 `max_tokens`：不要无限制输出，给明确的上限。大部分场景 1024 或 2048 就够用。
- 合理管理上下文：如果是多轮对话，只保留最近 3-5 轮的核心内容，舍弃早期无用的历史。千聚 API 本身没有特殊限制，但你自己做缓存和过滤能省很多。
- 使用 `temperature` 控制：对标准化任务（比如代码补全、文案改写）用 0.2-0.5，不需要太发散。

一个常见的例子：给模型传了 50k 的上下文，但真正用到的只有 5k。多出来的 45k 是白花钱。用千聚调用时，自己做好内容修剪，一个月轻松省下 30%-50%。

### 第三步：批量与缓存

如果你每天有大量重复请求（比如翻译、分类、摘要），不要每次都直接调用 GLM-4.5。

先建一个简单的缓存层：

python
import hashlib
import redis
from openai import OpenAI

client = OpenAI(
    api_key="your_key",
    base_url="https://www.qianjuai.com/v1"
)
cache = redis.Redis()

def cached_glm(messages: list, cache_ttl=3600):
    # 用消息内容生成唯一 key
    key = hashlib.md5(str(messages).encode()).hexdigest()
    cached = cache.get(key)
    if cached:
        return cached.decode()
    
    response = client.chat.completions.create(
        model="GLM-4.5",
        messages=messages
    )
    result = response.choices[0].message.content
    # 写入缓存
    cache.setex(key, cache_ttl, result)
    return result


这样相同的输入直接走缓存，不走 API，不花一分钱。高频场景下，这个策略能省掉至少一半费用。

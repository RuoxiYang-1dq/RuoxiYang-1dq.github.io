## o3-mini Java 接入实操

光说不练是假把式。下面直接上一段最纯粹、不加壳的Java代码，演示如何通过千聚调用o3-mini。

这个例子用了Java原生的 `HttpClient` 和 `JSON` 处理，不依赖任何第三方的AI SDK。意思很清楚：你的项目但凡有点年头，这套代码复制进去就能跑，零依赖冲突。

import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;
import java.time.Duration;

public class QianJuO3MiniDemo {

    // 全局配置
    private static final String API_KEY = "sk-你的千聚API密钥"; // 替换为你的真实密钥
    private static final String API_URL = "https://www.qianjuai.com/v1/chat/completions";

    public static void main(String[] args) throws Exception {
        // 构建请求体JSON
        String jsonBody = """
                {
                  "model": "o3-mini",
                  "messages": [
                    {"role": "system", "content": "你是一位精通Java后端开发的资深工程师。"},
                    {"role": "user", "content": "请用Java写一个实现LRU缓存的高性能代码。"}
                  ]
                }
                """;

        // 创建HttpClient
        HttpClient client = HttpClient.newBuilder()
                .connectTimeout(Duration.ofSeconds(30))
                .version(HttpClient.Version.HTTP_2)
                .build();

        // 构建请求
        HttpRequest request = HttpRequest.newBuilder()
                .uri(URI.create(API_URL))
                .header("Content-Type", "application/json")
                .header("Authorization", "Bearer " + API_KEY)
                .timeout(Duration.ofSeconds(120))
                .POST(HttpRequest.BodyPublishers.ofString(jsonBody))
                .build();

        // 发送请求并处理响应
        HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());

        System.out.println("状态码: " + response.statusCode());
        System.out.println("响应结果: " + response.body());
    }
}

- 模型的切换只体现在 `model` 字段上，今天写 `o3-mini`，明天改成 `deepseek-r1`，代码其他地方不用动。
- 请求地址只要写对一次，就能通吃所有模型。
- 响应格式完全兼容OpenAI标准，你现有的JSON解析逻辑可以直接复用。

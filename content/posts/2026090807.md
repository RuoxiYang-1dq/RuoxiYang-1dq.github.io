## 常见报错及解决方案（一次过不报错的关键）

虽然我说“一次过不报错”，但如果你在接入过程中遇到了问题，绝大部分都是下面三个原因造成的。看完就能自己解决：

**报错 1：`401 Authentication Error`**
- **原因**：API Key 写错了，或者复制时多带了空格/换行符。
- **解决**：重新从千聚控制台复制 Key，粘贴时确认前后没有多余字符。

**报错 2：`404 Not Found`**
- **原因**：模型名写错了。比如写成 `gpt-4.1-mini` 但实际千聚支持的模型名可能是 `gpt-4.1-mini-2026-04`。
- **解决**：去千聚文档页查看最新支持的模型列表，确认模型名大小写和末尾日期是否正确。

**报错 3：`429 Rate Limit`**
- **原因**：短时间内请求太频繁，触发了平台限速。
- **解决**：降低请求频率，加几行 `time.sleep()`，或者升级到更高并发权限的分组。

**报错 4：`Connection Timeout`**
- **原因**：网络环境不稳定，或你的机器 DNS 解析有问题。
- **解决**：检查是否挂了代理（如果有，先关掉），或者将 `base_url` 改为 `https://www.qianjuai.com/v1` 试试（去掉 https 不行就加上）。

如果你接入后依然报错，99% 以上的问题都可以通过检查 API Key 有效性、模型名是否存在、网络是否直连这三步解决。千聚的官方文档里也有非常详细的客户端配置截图教程，直接对照操作就行。

### 坑 2：不开启流式输出，白花冤枉钱
很多人写示例时，默认不设置 `stream: true`。但 Qwen-Turbo 不支持大段非流式输出？不，它支持，但非流式输出会按 **完整输出内容** 计费，即使你只用了前半句。

**省钱配置：**
javascript
async function callWithStream() {
  const stream = await client.chat.completions.create({
    model: 'qwen-turbo',
    messages: [
      { role: 'user', content: '写一篇5000字的文章' },
    ],
    stream: true, // 关键！流式输出
    max_tokens: 1000, // 限制输出长度
  });

  for await (const chunk of stream) {
    process.stdout.write(chunk.choices[0]?.delta?.content || '');
  }
}


你看见没有？加上 `stream: true` 后，你可以在输出过程中手动 `break` 循环，强制停止请求。这意味着：如果模型生成的内容你不需要了，可以立刻掐断，后续的 token 不扣费。

### 最狠省钱术：限制输出 Token，并调低温度

通过参数设置，你可以强制模型只输出你认为必要的内容。

javascript
async function cheapCall() {
  const response = await client.chat.completions.create({
    model: 'qwen-turbo',
    messages: [
      { role: 'system', content: '你是一个只会输出数字的AI' },
      { role: 'user', content: '1+1等于几？' },
    ],
    max_tokens: 10,    // 输出最多10个token，防废话
    temperature: 0.1,  // 低温度，让模型更确定、不瞎编
  });
  console.log(response.choices[0].message.content); // 输出: 2
}


通过 `max_tokens: 10`，即使你问再复杂的问题，输出最多只扣几毛钱。如果模型想长篇大论，也被强行截断。这招对于测试和快速验证逻辑极其好用。

## 实战：用成本最低的方式做一个 Qwen-Turbo 对话机器人

接下来，我手把手带你写一个能上线、还省钱的 Node.js 对话服务。

### 场景：客服问答
我们只允许用户问三个以内的字，然后强制限制输出。这样可以避免用户刷 token。

javascript
const express = require('express');
const app = express();
app.use(express.json());

const client = new OpenAI({
  baseURL: 'https://www.qianjuai.com/v1',
  apiKey: process.env.QIANJU_API_KEY,
});

app.post('/ask', async (req, res) => {
  const userMessage = req.body.message;
  
  // 强制用户输入短文本，防止恶意输入
  if (userMessage.length > 100) {
    return res.status(400).json({ error: '消息太长了，省点钱吧' });
  }

  const stream = await client.chat.completions.create({
    model: 'qwen-turbo',
    messages: [
      { role: 'system', content: '你是客服小助手，每次回复不超过30字。' },
      { role: 'user', content: userMessage },
    ],
    max_tokens: 60, // 30个中文字符大约50token
    temperature: 0.2,
    stream: true,
  });

  res.setHeader('Content-Type', 'text/plain');
  for await (const chunk of stream) {
    res.write(chunk.choices[0]?.delta?.content || '');
  }
  res.end();
});

app.listen(3000);
console.log('省钱API服务启动在 http://localhost:3000');


你看，通过系统指令强迫模型简短答，再通过 `max_tokens` 强制执行，一次请求的成本可能只有 0.01 元。比发一条短信还便宜。

## 常见报错大全 & 避坑方法

**报错 1: 401 Unauthorized**
- 原因：API Key 填错了，或者只用了千聚官网的 Key，但没有在代码里正确赋值。
- 解决：确保 `apiKey` 变量从 `.env` 文件读入。在千聚后台复制 Key 时，注意别复制到多余空格。

**报错 2: 402 Payment Required**
- 原因：余额不足。[千聚api聚合站](https://www.qianjuai.com/)的计费是实时扣费的，你没充值或者额度用完了就会这样。
- 解决：最低充 1 元即可。去 [千聚官网充值](www.qianjuai.com)。

**报错 3: Token 超限，模型直接静默报错**
- 原因：你的 `max_tokens` 设少了，但实际上模型尝试生成的内容很长。
- 解决：观察日志，逐步调高 `max_tokens` 或者按实际需求切分输入内容。

## 这份Python脚本，就是你的模型“万能遥控器”

下面这个示例，就是解决以上所有痛点的一套代码。它核心就干一件事：**用一个函数，切换不同模型。**

python
import os
from openai import OpenAI

# 核心配置：只需要在千聚后台获取一次 API Key
qianju_api_key = "sk-your-qianju-api-key-here"
qianju_base_url = "https://www.qianjuai.com/v1"  # 强制使用此链接

# ---- 模型映射表 ----
# 豆包、DeepSeek、通义千问在千聚上的具体模型名称
MODEL_MAPPING = {
    "豆包": "ep-20250219-doubao-pro-32k",   # 示例ID，以实际为准
    "DeepSeek": "deepseek-chat",           # DeepSeek-V3
    "通义千问": "qwen-plus",              # 通义千问Plus
}

# ---- 单一调用函数 ----
def call_ai_model(model_key: str, user_prompt: str) -> str:
    """
    根据传入的模型键名，自动切换并调用千聚上的对应模型。
    """
    model_name = MODEL_MAPPING.get(model_key)
    if not model_name:
        return f"错误：未找到模型 '{model_key}'，请检查MODEL_MAPPING。"

    client = OpenAI(
        api_key=qianju_api_key,
        base_url=qianju_base_url
    )

    try:
        response = client.chat.completions.create(
            model=model_name,
            messages=[
                {"role": "user", "content": user_prompt}
            ],
            temperature=0.7,
            max_tokens=1024
        )
        return response.choices[0].message.content
    except Exception as e:
        return f"调用失败：{e}"

# ---- 一句话切换三大模型 ----
if __name__ == "__main__":
    user_input = "请用20字以内概括今天的天气情况。"

    print("使用【豆包】的回答：")
    print(call_ai_model("豆包", user_input))

    print("\n使用【DeepSeek】的回答：")
    print(call_ai_model("DeepSeek", user_input))

    print("\n使用【通义千问】的回答：")
    print(call_ai_model("通义千问", user_input))


**这段代码干了什么？**

-   它定义了一个**模型映射字典** `MODEL_MAPPING`，把豆包、DeepSeek、通义千问在千聚上的模型ID存起来。
-   核心函数 `call_ai_model`，接收一个键（比如 "豆包"），自动去字典里找模型ID，然后用统一的`base_url`和`api_key`调用。
-   在主程序里，我们**一句话**切换模型：把 `model_key` 从 "豆包" 改成 "DeepSeek" 或者 "通义千问"，就实现了模型的无缝切换。

**错误四：未正确处理请求头或参数缺失**

有些开发者在自定义请求时，随意修改了请求头格式，或者忽略了某些必要参数。

- **症状：** 调用时返回 400 Bad Request 错误，或者模型返回奇怪的非正式输出。
- **原因分析：** 常见的坑包括：没有设置 `Content-Type: application/json`、没有正确传递 `Authorization` 字段、缺少 `max_tokens` 或 `temperature` 参数等。特别是当使用流式输出（stream）时，如果参数配置不当，会导致模型挂起或崩溃。
- **解决方案：** 返回 OpenAI 的标准请求格式。对于 GPT-4.1nano，建议至少提供 `model` 和 `messages` 字段，再根据需求设置 `max_tokens` 和 `temperature`。例如：
  json
  {
    "model": "gpt-4.1-nano",
    "messages": [{"role": "user", "content": "你好"}],
    "max_tokens": 512,
    "temperature": 0.7
  }
  
  保持 API Key 以 `Bearer YOUR_API_KEY` 格式放在请求头中。
